# Runs 🚀

This folder contains the active run entrypoints for MIRA simulation and stress-test experiments.

Canonical working directory for environment setup and service startup: `HospitalAgent/` (repository root).

## Current Structure

```text
runs/
├── README.md
├── __init__.py
├── run.py                               # core simulation runner used by baseline + most bias runs
├── run_optional_admission.py            # simulation runner for optional-admission experiments
├── run_with_sex_bias.py                 # sex-bias variant used by run_simulation_bias.ipynb
├── run_simulation.ipynb                 # baseline entrypoint
├── run_simulation_bias.ipynb            # bias entrypoint (includes sex-bias flow)
├── run_simulation_optional_admission.ipynb
└── run_leakage_and_adversarial.ipynb
```

## Canonical Run Notebooks

- `run_simulation.ipynb`
  - Uses `run.py`.
  - Writes baseline outputs to `src/raw/evaluable_outputs/baseline/EVALUABLE_OUTPUTS`.
- `run_simulation_bias.ipynb`
  - Uses `run.py` and `run_with_sex_bias.py`.
  - Writes outputs to `src/raw/evaluable_outputs/bias/EVALUABLE_OUTPUTS_BIAS_*`.
- `run_simulation_optional_admission.ipynb`
  - Uses `run_optional_admission.py`.
  - Writes outputs to `src/raw/evaluable_outputs/optional_admission/EVALUABLE_OUTPUTS_OPTIONAL_ADMISSION`.
- `run_leakage_and_adversarial.ipynb`
  - Leakage/adversarial analysis notebook.
  - Uses baseline/human evaluable outputs as input and writes local analysis artifacts.

## Notes

- Runtime in this folder is evaluation-only (`EVALUATION_MODE=True`); interactive mode has been removed.
- Before running simulations, ensure:
  - Diagnosis datasets are built (`src/dataset/README.md`).
  - `src/resources/pancreatic_cancer_info.json` exists (generated by `src/notebooks/extract_pancreatic_cancer_info.ipynb` when needed).
  - Local FHIR backend is running (`src/backend/README.md`).
  - Local Qdrant is running and procedure DB is prepared (`HospitalAgent/README.md`, Steps 5-6).
- Open run notebooks from `src/runs/` so local notebook imports resolve as written.
- Notebooks are parameterized for selective/smoke runs by default (for example small `MAX_SAMPLES`, narrowed `DATASET_NAMES`, and explicit guards such as `break` or `RUN_CELL=False` in analysis flows). Review the first parameter cells before full runs.
