# Raw Data Layout

Runtime/generated artifacts and local input data used by dataset extraction, runs, and evaluations.

Canonical working directory for all paths/commands below: `HospitalAgent/` (repository root).

## Structure

```text
raw/
├── derived/
│   └── diagnosis_datasets/                 # generated diagnosis datasets (parquet/json)
├── evaluable_outputs/                      # generated run outputs used by evaluations
│   ├── baseline/
│   ├── bias/
│   ├── optional_admission/
│   └── human_bc/
├── inputs/
│   └── optional_admission/                 # edited Excel inputs for optional-admission dataset build
├── runtime/
│   └── qdrant/main/                        # local Qdrant storage (will be populated once Qdrant is launched)
└── archive/
    └── results_backup/                     # optional local backup location
```

## How folders are populated

- `derived/diagnosis_datasets/`:
  - This folder can be empty in a clean/publication package.
  - Fill it by running:

```bash
uv run --project src python src/dataset/make_dataset.py
```

- Optional-admission datasets are generated by:

```bash
uv run --project src python src/dataset/make_admission_datasets.py
```

- `evaluable_outputs/`:
  - Filled by run notebooks in `src/runs/` (baseline, bias, optional admission).
  - Evaluation notebooks in `src/evaluations/` read from these outputs.

- `inputs/optional_admission/`:
  - Contains the curated `.xlsx` source inputs required by `make_admission_datasets.py`.
  - Expected subtrees:
    - `src/raw/inputs/optional_admission/pneumonia/**/*.xlsx`
    - `src/raw/inputs/optional_admission/pe/**/*.xlsx`

- `runtime/qdrant/main/`:
  - Local vector DB storage used by Qdrant and the procedure retrieval pipeline.
  - Created/filled when running Qdrant + procedure DB generation.

## Notes

- Root `.gitignore` excludes most of `src/raw/` (generated/local artifacts), with `src/raw/README.md` intentionally tracked.
