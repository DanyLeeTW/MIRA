# Evaluations

This folder contains the active evaluation pipeline for MIRA and human outputs.
It also contains helper scripts used by the evaluation notebooks.

Canonical working directory for evaluation notebooks/scripts: `HospitalAgent/src/evaluations/`.

Input data is expected from:

- `src/raw/evaluable_outputs/baseline/EVALUABLE_OUTPUTS` (baseline simulation outputs)
- `src/raw/evaluable_outputs/bias/EVALUABLE_OUTPUTS_BIAS_*` (optional bias runs)
- `src/raw/evaluable_outputs/human_bc/EVALUABLE_OUTPUTS_HU_BC` (optional human baseline outputs)

## Current Folder Structure

```text
evaluations/
├── __init__.py
├── README.md
├── eval_config.py
├── evaluate.py
├── objectives.py
├── preprocess.py
├── evaluations_MIRA.ipynb
├── evaluations_HUMANS.ipynb
├── evaluate_answer_consistency.ipynb
├── split_by_diagnosis.ipynb
├── MIRA_EVALUATED_OUTPUTS/                  # generated by evaluations_MIRA.ipynb
├── MIRA_EVALUATED_OUTPUTS_*/                # generated for bias runs (e.g., recency_bias, healthy_bias, anxiety_bias, french_bias, german_bias, sex)
└── ...                                      # optional local analysis artifacts
```

## Notes

- Figure data generation run entrypoints live in `src/runs/` (for example `src/runs/run_leakage_and_adversarial.ipynb`).
- Open evaluation notebooks from `HospitalAgent/src/evaluations/` so local imports like `from preprocess import ...` resolve as written.

## Evaluation Steps

1. Generate baseline simulation outputs first (for example via `src/runs/run_simulation.ipynb`), so `src/raw/evaluable_outputs/baseline/EVALUABLE_OUTPUTS` exists.
2. Run `evaluations_MIRA.ipynb`.
   - This notebook uses functions from `objectives.py` and `preprocess.py`.
   - Outputs are written to `MIRA_EVALUATED_OUTPUTS/`.
   - If bias inputs exist, this notebook also writes `MIRA_EVALUATED_OUTPUTS_<bias>/` folders.
3. Optionally run `evaluations_HUMANS.ipynb` for human baselines.
   - Expects input under `src/raw/evaluable_outputs/human_bc/EVALUABLE_OUTPUTS_HU_BC`.
   - Outputs are written to `HUMAN_EVALUATED_OUTPUTS/`.
4. Optionally run `split_by_diagnosis.ipynb` for diagnosis-based splits (useful for some downstream analyses).
